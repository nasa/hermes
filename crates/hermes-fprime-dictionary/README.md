# hermes-fprime-dictionary

A Rust library and CLI tool for converting F' (FPrime) JSON dictionaries to Hermes protobuf format.

## Library Usage

Add this to your `Cargo.toml`:

```toml
[dependencies]
hermes-fprime-dictionary = { path = "path/to/hermes-fprime-dictionary" }
```

Example:

```rust
use hermes_fprime_dictionary::parse_fprime_json_dictionary;

let json = r#"{
    "metadata": {"deploymentName": "MyMission"},
    "commands": [],
    "events": [],
    "telemetryChannels": []
}"#;

let dictionary = parse_fprime_json_dictionary(json).unwrap();
```

## CLI Usage

Install the binary:

```bash
cargo install --path crates/hermes-fprime-dictionary
```

Convert an F' JSON dictionary to Hermes protobuf format:

```bash
hermes-fprime-dictionary dict.json > out.pb
```

### Example

```bash
# Convert an F' dictionary
hermes-fprime-dictionary /path/to/fprime-dictionary.json > hermes-dictionary.pb

# View help
hermes-fprime-dictionary --help
```

## Features

- Parses F' JSON dictionaries (generated by FPP or fprime-tools)
- Converts to Hermes protobuf-based Dictionary format
- Supports:
  - Type definitions (enums, structs, arrays, aliases)
  - Commands with arguments
  - Events with severity levels and format strings
  - Telemetry channels
  - Parameters
  - Telemetry packet sets

## Dictionary Structure

The resulting Dictionary contains:

- **Namespaces**: Two namespaces are created:
  - `""` (empty string): Contains all commands, events, telemetry channels, parameters, and types
  - `"telemetry-packets"`: Contains telemetry packet definitions (if any)
- **Metadata**: Additional dictionary metadata like spacecraft ID

## Replaces

This Rust implementation replaces the TypeScript version at `src/scripts/fprime-json-dictionary.ts` with a native, standalone binary that doesn't require Node.js.
